P1 Product Overview

DocReader is a personal Android application that provides a distraction-free, Kindle-style reading experience for Google Docs stored in the user's Google Drive. It combines comfortable long-form reading with high-quality voice narration, designed for a single user who wants to read or listen to their own documents without any data being stored on the device.

The app is intentionally simple: sign in, browse your docs, read or listen, sign out. Nothing is saved between sessions.

P2 Goals & Success Metrics

Goals

  • Make reading long Google Docs comfortable on a phone screen
  • Provide a voice reading experience that sounds natural, not robotic
  • Ensure zero data persistence — every session starts and ends clean
  • Keep the UI simple enough to use without learning

Mayuresh — The Primary (and only) User

  • Wants to read long docs on the phone during commute, travel, or downtime

P4 Functional Requirements

P4.1 Authentication

P4.1.1
Google Sign-In
The app shall provide a Google Sign-In flow using Android Credential Manager. The user must authenticate with their Google account before accessing any content.
P4.1.2
Scope of access
The app shall request only the minimum OAuth scopes required:
  • https://www.googleapis.com/auth/drive.readonly — read Drive file list
  • https://www.googleapis.com/auth/documents.readonly — read document content
P4.1.3
No persistent login
The app shall not store authentication tokens to disk (no SharedPreferences, no database, no files). The token lives in memory for the duration of the session only. On every app launch, the user must sign in again.
P4.1.4
Sign-out
A sign-out action shall be available from the Drive browser screen. Signing out immediately clears all in-memory state and returns the user to the login screen.

P4.2 Session Management

P4.2.1
Inactivity timeout
If the user has not interacted with the app (scroll, tap, swipe) for 5 consecutive minutes, the session shall expire automatically, all in-memory data shall be cleared, and the user shall be returned to the login screen.
P4.2.2
Timer reset on interaction
Any user interaction (tap, scroll, swipe) shall reset the 5-minute inactivity countdown to zero.
P4.2.3
Background behaviour
When the app moves to the background (user switches to another app) and voice reading is not actively playing, the 5-minute inactivity countdown shall continue running. If the countdown expires while in the background, the session shall be cleared so that returning to the app shows the login screen.
P4.2.4
Voice reading exemption
While voice reading is actively playing, the inactivity timer shall be suspended. The sleep timer (P4.6) governs session end during voice reading.
P4.2.5
App close / destroy
When the app is closed or destroyed by the OS, all session data shall be cleared immediately.

P4.3 Drive Browser

P4.3.1
Root folder view
On successful login, the app shall display the root of the user's Google Drive showing folders and Google Docs owned by the user. Shared documents shall not be shown.
P4.3.2
Folder navigation
Tapping a folder shall navigate into it and display its contents. A breadcrumb trail shall show the current path (e.g. My Drive › Work › Notes). Tapping any breadcrumb item shall navigate directly to that level.
P4.3.3
Back navigation
A back arrow shall appear when inside a subfolder, navigating up one level. The Android system back gesture/button shall also navigate up. Pressing back from the root folder shall sign the user out.
P4.3.4
Search
A search icon shall open a search bar. Searching queries across ALL folders in the user's Drive (not just the current folder). Results shall show only Google Docs (not folders). Clearing the search returns to the folder view that was active before searching.
P4.3.5
Open a document
Tapping a Google Doc in the list shall fetch the document content and open the Reader screen.
P4.3.6
Sort order
Items shall be sorted: folders first (alphabetical), then documents (alphabetical). Search results shall be sorted alphabetically.

P4.4 Document Reader

P4.4.1
Structured rendering
The reader shall render the document preserving its structure: headings (H1–H6 with visual hierarchy), paragraphs, bullet lists, and numbered lists. Images and tables are out of scope for v1 (see P7).
P4.4.2
Scroll mode
In scroll mode, the entire document is rendered as a continuous vertical scroll. This is the default reading mode.
P4.4.3
Paginated mode
In paginated mode, the document is divided into pages that fit the screen. The user navigates by swiping left/right or tapping left/right edges. The current page number and total are shown.
P4.4.4
Mode toggle
A toggle button in the toolbar shall switch between scroll and paginated mode at any time. Reading position shall be preserved on toggle (the same part of the document remains visible).
P4.4.5
No reading history
The app shall not save reading position, recently opened documents, or any other reading state. Every document opens from the beginning.

P4.5 Voice Reading

P4.5.1
TTS engine
Voice reading shall use the Google Cloud Text-to-Speech API with Neural2 voices. The API key is stored in the app's secret resource file (not in code or committed to git).
P4.5.2
Voice selection
The user shall be able to choose from a curated list of high-quality Neural2 voices across English accents (US, UK, Australian, Indian). The list shall show the voice name, accent, and gender.
P4.5.3
Playback controls
The voice control panel shall provide: Play / Pause, Stop, Skip to next section, Skip to previous section.
P4.5.4
Speed control
The user shall be able to adjust reading speed from 0.5× to 2.0× in 0.25× increments.
P4.5.5
Progress indicator
A progress bar and section counter (e.g. "Section 3 of 12") shall be shown during voice reading.
P4.5.6
No audio files on disk
TTS audio shall be streamed and played entirely from memory. No audio files shall be written to device storage at any point.
P4.5.7
Text chunking
Documents shall be split into chunks of ≤ 4000 characters at sentence boundaries before sending to the TTS API. The next chunk shall be pre-fetched while the current one is playing to minimise gaps.

P4.6 Sleep Timer

P4.6.1
Sleep timer options
The user shall be able to set a sleep timer with the following preset options: 15 minutes, 30 minutes, 45 minutes, 60 minutes, or a custom duration (entered in minutes).
P4.6.2
Timer behaviour
When the sleep timer fires: voice reading shall stop, all session data shall be cleared, and the user shall be returned to the login screen.
P4.6.3
Timer display
While a sleep timer is active, the remaining time shall be displayed in the voice control panel (e.g. "Sleep: 23m left").
P4.6.4
Cancel timer
The user shall be able to cancel an active sleep timer. Cancelling resumes normal inactivity-based session management.

P4.7 Reader Settings

P4.7.1
Font size
The user shall be able to increase or decrease the body text font size in 1sp steps, within the range 12sp–32sp. Heading sizes scale proportionally.
P4.7.2
Font family
The user shall be able to choose between three font families: Serif, Sans-serif, and Monospace.
P4.7.3
Background theme
The user shall be able to choose between three background themes: Light (white), Sepia (warm cream), and Dark (near-black with light text).
P4.7.4
Settings not persisted
Reader settings shall not be saved between sessions. Each new session starts with default settings (18sp, Serif, Light theme, Scroll mode).

P6 Privacy & Security

IDRequirement
P6.1No data written to local storage (SharedPreferences, Room, files) at any time
P6.2Auth tokens held in memory only; cleared immediately on session end
P6.3TTS audio held in memory only; never written to device storage
P6.4API keys stored in a gitignored resource file; never committed to source control
P6.5OAuth scopes limited to read-only access to Drive and Docs
P6.6No analytics, crash reporting, or third-party SDKs that transmit data
P6.7No reading history, bookmarks, or recently opened list retained between sessions

P7 Out of Scope (v1)

  • Images and embedded media within documents
  • Tables
  • Comments and suggestions
  • Shared documents (only owned documents are shown)
  • Offline reading / caching
  • Bookmarks or reading position persistence
  • Multiple user accounts
  • Play Store distribution
  • Landscape-optimised layout
  • Word-level highlight during voice reading
  • Background audio service (minimised app playback)

P8 Future Considerations

  • Upgrade TTS to ElevenLabs or Google Studio voices for more natural prosody
  • Word or sentence highlight during voice reading
  • Background audio playback (foreground service)
  • Reading position persistence (opt-in, with explicit user consent)
  • Support for shared documents
  • Basic table rendering
  • Custom sleep timer sound (fade out)
